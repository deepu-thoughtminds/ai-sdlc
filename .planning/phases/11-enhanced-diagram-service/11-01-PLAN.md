<objective>
Enhance `backend/services/drawio_service.py` in place to satisfy ARCHGEN-04 and CONFPUB-02.

The current service generates generic rounded-rectangle cells with unstyled (no arrowhead) edges. This plan adds:

1. **Typed component shapes** (per D-01 design decision): keyword-based routing so "database"/"db"/"storage" components get cylinder style, "external"/"client"/"api gateway" components get a callout/rhombus shape, and all others keep the rounded-rectangle service style.
2. **Directional edges** (per D-02): `endArrow=block` added to edge cell style attribute.
3. **`validate_xml(xml: str) -> bool`** (per D-03): uses `xml.etree.ElementTree.fromstring()` — returns `True` if XML parses without error, `False` on `ET.ParseError`.
4. **`generate_viewer_url(xml: str) -> str`** (per D-04): URL-encodes the XML with `urllib.parse.quote()` and returns `https://app.diagrams.net/?xml=<encoded>`.

`generate_diagram()` signature remains `(title, components, connections) -> str` for backward compatibility; `architecture_pipeline.py` is NOT modified in this plan — its callers gain access to the new helpers by importing them directly when needed.

Purpose: Architects can open diagrams in diagrams.net without the Confluence draw.io plugin; the pipeline can degrade gracefully on malformed XML.
Output: Enhanced `drawio_service.py` + expanded test suite.
</objective>
