<objective>
Replace the single-template, always-create `ConfluenceClient.publish_architecture()` (built in Phase 4) with a version that: (1) renders one of two HTML templates depending on a branch flag — text-only or diagram+components — with all LLM-generated text HTML-escaped before interpolation; (2) for diagram pages, embeds the raw mxGraph XML in a `<pre class="drawio-xml">` block plus an `<a href="...">` link to the diagrams.net viewer URL (consuming `generate_viewer_url` from Phase 11's `drawio_service.py`); (3) searches Confluence for an existing page titled `Architecture: {issue_key}` before creating one, updating in place if found; (4) catches every exception from search/create/update and returns `""` so the caller's existing graceful-degradation path (Jira comment with inline text, no URL) keeps working unchanged.

Purpose: This phase only changes `confluence_client.py`'s internals and its public signature — `architecture_pipeline.py` is rewired to call the new signature in Phase 13. This phase's job is the Confluence HTML templates, escaping, and page lifecycle (search → update-or-create), not diagram generation (done in Phase 11) or pipeline orchestration (done in Phase 13).
Output: Updated `backend/services/confluence_client.py` with new template builders, `find_page`/`update_page` methods, a new `publish_architecture` signature accepting structured section text plus a diagram/components branch flag; updated `backend/tests/test_confluence_client.py` covering both templates, escaping, idempotent find-or-update, and graceful degradation.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| LLM output → Confluence HTML body | architecture_pipeline (Phase 13) will pass LLM-generated summary/approach/component-breakdown/integration-points/key-decisions/risks text into this client; that text is untrusted and must never be interpolated as raw HTML |
| confluence_client → Confluence Cloud REST API | outbound HTTPS calls carrying decrypted bearer/basic-auth token; response JSON (search results, page ids, version numbers) is treated as structured data, not re-interpreted as HTML |
| drawio_service → confluence_client | raw mxGraph XML and viewer URL cross this boundary; XML is pre-escaped/pre-validated by drawio_service (Phase 11) and must not be double-escaped or have its structure broken by this phase's templates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-12-01 | Tampering | `_render_text_only_template` / `_render_diagram_template` | mitigate | All LLM-generated text params (summary, approach, component_breakdown, integration_points, key_decisions, risks) pass through `_escape()` (html.escape with quote=True) before string interpolation — prevents stored XSS in Confluence storage-format HTML rendered later in a browser |
| T-12-02 | Tampering | raw mxGraph XML embedded in `<pre class="drawio-xml">` | accept | XML is pure text content inside a `<pre>` block (not executed); drawio_service.py's `_escape_xml` already escapes XML metacharacters at generation time (T-04-04 carried forward); confluence_client does not re-escape to avoid corrupting the diagram source the architect copies out |
| T-12-03 | Information Disclosure | `find_page`/`update_page`/`create_page` | accept | All three reuse the existing `_headers()`/Basic-auth pattern (T-04-02/T-04-06); decrypted token never logged; carried forward unchanged from Phase 4 |
| T-12-04 | Denial of Service | `find_page` CQL search | mitigate | `limit=1` param on the search request bounds response size regardless of how many pages match the title query |
| T-12-05 | Tampering | duplicate-page race (two pipeline runs hit publish_architecture concurrently for the same issue_key) | accept | Confluence's own optimistic version-locking (PUT with explicit version number) will reject a stale update with a 409, which falls into the existing catch-all except → returns "" — graceful degradation, not a crash; acceptable since CONFPUB-04 already requires this exact behavior on any publish failure |
| T-12-06 | Tampering | npm/pip/cargo installs | accept | No new packages introduced in this phase; `html` is a Python stdlib module, not a third-party dependency |
</threat_model>
