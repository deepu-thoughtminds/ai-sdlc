    Edit the architecture branch of handle_jira_comment() in backend/routers/webhook.py. The branch is currently at the elif block where mention_result.stage == "architecture".

    Replace the current branch (which just calls asyncio.create_task directly) with:

    1. Query for existing active pipeline:
       existing = db.query(PipelineState).filter(
           PipelineState.ticket_key == event.issue.key,
           PipelineState.stage == "architecture",
           PipelineState.status != "failed",
       ).first()
       if existing is not None:
           logger.info("Architecture pipeline already active for issue %s (state_id=%s) — ignoring duplicate", event.issue.key, existing.id)
           return {"status": "received", "action": "ignored", "reason": "duplicate_pipeline"}

    2. Create PipelineState row with status="running" BEFORE scheduling the background task. This is what the idempotency guard checks — the row must exist before asyncio.create_task is called so a near-simultaneous second webhook sees it:
       state_row = PipelineState(
           project_id=project.id,
           ticket_key=event.issue.key,
           stage="architecture",
           status="running",
       )
       db.add(state_row)
       db.commit()

    3. Then call asyncio.create_task(architecture_pipeline.run(...)) as before.

    NOTE: The new architecture_pipeline.run() (Plan 01) also creates a PipelineState row at its start. Since the webhook now creates the row first, the pipeline's internal row creation must be skipped OR the pipeline must re-use/update the existing row. To keep coupling minimal: the webhook creates the row, the pipeline's run() should query for the most recent "running" row for the ticket+stage and update it (instead of inserting a new row). If none found (unlikely), it falls back to creating one. This avoids duplicate rows. Document this in the action of Plan 01 if not already handled — for this plan, the webhook creates the guard row and the pipeline re-uses it.

    Do NOT change any other branch (describe, assign, approval detector).
