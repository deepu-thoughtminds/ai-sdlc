Create `backend/services/intent_router.py`. The module classifies a free-text mention phrase (everything after `@jarvis`) into a structured intent using `route_request('classify', prompt)` from `services.llm_router`.

**IntentResult dataclass:**
```
action: str       # one of VALID_ACTIONS or "unknown"
confidence: float # 0.0 to 1.0
entities: dict    # e.g. {"user": "@alice"}, {"target": "story description"}, {} for most
```

**VALID_ACTIONS set** (module-level constant):
`{"describe", "architecture", "start_coding", "merge_pr", "assign", "approve"}`

**classify_intent(mention_text: str) -> IntentResult | None:**

Build a structured classification prompt. The prompt must instruct the LLM to respond ONLY with a JSON object in the format `{"action": "<action>", "confidence": <float>, "entities": {<key>: <value>}}`. The prompt must list the valid actions with natural-language descriptions so the model understands mapping:
- `describe` — elaborate/describe the ticket, write description
- `architecture` — generate architecture, design system, draw diagram
- `start_coding` — start coding, begin implementation, write code, dev
- `merge_pr` — merge pr, merge the PR, merge branch, merge to main
- `assign` — assign to someone, assign @name
- `approve` — approve story description, approve architecture

The prompt must include the input mention text and instruct the model: if the input does not match any known action or is very ambiguous, return `{"action": "unknown", "confidence": 0.0, "entities": {}}`.

Call `route_request('classify', prompt)` — this is in HEAVY_STAGES already. Add `'classify'` to `HEAVY_STAGES` in `llm_router.py` if not already present (check first with grep; it is already there per line 30 of llm_router.py — do NOT duplicate).

Parse the LLM response content as JSON (use `json.loads`). Handle all exceptions (JSONDecodeError, KeyError, httpx errors) with a try/except that logs a warning and returns None — never raises.

After parsing: if `action` is not in VALID_ACTIONS or action is "unknown" or confidence < 0.5, return None. Otherwise return `IntentResult(action=action, confidence=confidence, entities=entities)`.

Threat mitigations:
- T-14-01 (Tampering): prompt only contains the mention text string — no credential values or sensitive data are interpolated into the LLM prompt.
- T-14-02 (Info Disclosure): LLM response JSON is never logged at INFO; log only the classified action at INFO.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Jira comment → intent classifier | Free-text from Jira comment body enters the LLM prompt. No user credentials cross this boundary — only comment text. |
| intent_router → freellmapi | LLM prompt sent to internal Docker network service (ai-sdlc-net). Response JSON parsed defensively. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | intent_router prompt construction | mitigate | Prompt contains only mention_text string; no credential values or issue tokens are interpolated |
| T-14-02 | Information Disclosure | LLM response logging | mitigate | LLM response content never logged at INFO; only classified action name logged |
| T-14-03 | Spoofing | @jarvis mention target check | mitigate | parse_mention() filters to mention_target.lower() == "jarvis" only — other agents ignored |
