<objective>
Add the read-path plumbing the dev pipeline needs: Hermes MCP client methods to fetch a Jira issue's comment history and a Confluence page's body content, the corresponding Hermes HTTP endpoints, the backend HTTP client wrappers that call them, and a pure-function utility that scans comment history for the most recent Confluence architecture page URL.

Purpose: DEVPIPE-01 requires the agent to search a ticket's comment history for the most recent Confluence architecture page URL and fetch that page's content. None of this read-path exists yet — Phase 15 only built the write-path (clone/codegen/PR). This plan builds every piece needed to go from "Jira issue_key" to "Confluence architecture page text" before Plan 02 wires it into the dev pipeline orchestrator.

Output: `hermes/mcp_client.py` (2 new methods), `hermes/server.py` (2 new endpoints), `backend/services/hermes_client.py` (2 new functions), `backend/services/confluence_url_finder.py` (new file), plus tests for all four.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend → hermes (HTTP) | backend calls hermes's internal /jira/* and /confluence/* endpoints over the docker-compose network |
| hermes → mcp-atlassian (MCP/HTTP) | hermes's MCP client calls mcp-atlassian tools with per-request credentials |
| Jira comment body → regex parser | find_latest_architecture_url parses untrusted Jira comment text (potentially attacker-influenced if a non-team-member can comment) for URL patterns |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Information Disclosure | hermes_client.get_comments / get_confluence_page_content | mitigate | Never log jira_token/confluence_token; only issue_key/page_id logged (T-09-01 convention carried forward) |
| T-16-02 | Denial of Service | confluence_url_finder.find_latest_architecture_url | accept | Regex is anchored/bounded (`[^\s]+` excludes whitespace, no nested quantifiers) — no catastrophic backtracking risk; comment bodies are already bounded to max_length=10000 at the Pydantic model layer (T-02-04) |
| T-16-03 | Tampering | hermes/server.py new GET /confluence/page/{page_id} endpoint | accept | Internal-network-only endpoint (not exposed externally), same trust model as existing /confluence/search; page_id is passed through to mcp-atlassian which validates it |
| T-16-04 | Denial of Service | hermes_client.get_comments returning unbounded comment list | accept | Same trust model as existing post_sprint_backlog — Jira-side pagination/limits apply upstream; no new unbounded surface introduced |
</threat_model>
