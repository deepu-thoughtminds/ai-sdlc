    Create backend/services/codebase_scan_service.py from scratch. The module's public surface is a single coroutine:

        async def run(github_repo: str, github_token: str, project_id: int, db: Session) -> None

    github_repo is the decrypted owner/repo slug (e.g. "acme/my-app") — NOT a full GitHub URL. Parse via:

        match = re.match(r"^([A-Za-z0-9_.-]+)/([A-Za-z0-9_.-]+)$", github_repo.strip())

    If match fails, log a warning at WARNING level and return early without raising. Do not log github_token (T-18-01).

    Module-level constants (define exactly as shown, per RESEARCH.md):

        GITHUB_API_BASE = os.environ.get("GITHUB_API_BASE", "https://api.github.com")

        SKIP_DIRS = frozenset({
            "node_modules", ".git", "__pycache__", ".venv", "venv", "env",
            "dist", "build", ".next", "out", "coverage", ".nyc_output",
            ".pytest_cache", ".mypy_cache", ".ruff_cache", "vendor", "migrations",
        })

        SKIP_EXTENSIONS = frozenset({
            ".png", ".jpg", ".jpeg", ".gif", ".svg", ".ico", ".webp",
            ".woff", ".woff2", ".ttf", ".eot", ".pdf", ".docx", ".xlsx",
            ".zip", ".tar", ".gz", ".lock", ".pyc", ".pyo", ".so",
            ".db", ".sqlite", ".sqlite3", ".min.js", ".min.css",
        })

        MAX_FILES = 25
        MAX_FILE_CHARS = 2000

    Internal helper functions (all module-private, prefixed _):

    _skip_path(path: str) -> bool — returns True if any path component is in SKIP_DIRS OR the extension is in SKIP_EXTENSIONS. Split path on "/" to get components; use pathlib.PurePosixPath(path).suffix for extension.

    _select_key_files(all_paths: list[str]) -> list[str] — given the flat list of blob paths from the Trees API, return up to MAX_FILES paths, applying SKIP_DIRS/SKIP_EXTENSIONS, in priority order:
      1. README.md / README.rst / README.txt (any location, case-insensitive filename match)
      2. docker-compose.yml, docker-compose.yaml, Dockerfile (root only — no "/" in path)
      3. pyproject.toml, requirements.txt, package.json (depth ≤ 2 = at most one "/" in path), Cargo.toml, go.mod (root)
      4. .env.example, .env.sample (root)
      5. Entry points at depth ≤ 1 (root or one dir deep): main.py, app.py, wsgi.py, asgi.py, manage.py, index.ts, index.js, main.go, main.rs
      6. .github/workflows/ — first 2 YAML files
      7. Top-level __init__.py (depth ≤ 2)
      8. src/index.ts, src/main.ts (exactly these paths)
      Fill remaining budget with any not-yet-selected blob paths that pass _skip_path.

    _build_ascii_tree(all_paths: list[str], max_depth: int = 3) -> str — build an ASCII directory tree from the flat path list. Cap each branch at max_depth components; represent collapsed deeper paths as "{dir}/...". Format:
      repo-root/
      ├── backend/
      │   ├── services/
      │   └── main.py
      └── README.md
    Sort entries: directories before files at each level; both groups sorted alphabetically. Use "├── " for non-last entries, "└── " for last. Prefix: "│   " for continued, "    " for completed. Skip paths whose components include a SKIP_DIRS entry.

    _detect_tech_stack(all_paths: list[str], manifest_contents: dict[str, str]) -> list[str] — return a list of "Language/Framework (source)" strings. Check presence signals (T-18-02 — paths treated as data):
      - "Python": any of pyproject.toml, setup.py, requirements.txt in all_paths
      - "FastAPI": "fastapi" (case-insensitive) in manifest_contents.get("pyproject.toml", "") or manifest_contents.get("requirements.txt", "")
      - "Django": "manage.py" in all_paths
      - "Node.js": any p.endswith("package.json") for p in all_paths
      - "Next.js": "\"next\"" in manifest_contents.get("package.json", "")
      - "React": "\"react\"" in manifest_contents.get("package.json", "")
      - "TypeScript": "tsconfig.json" in all_paths
      - "Docker": "Dockerfile" in all_paths or "docker-compose.yml" in all_paths
      - "GitHub Actions": any p.startswith(".github/workflows/") for p in all_paths
      - "Go": "go.mod" in all_paths
      - "Rust": "Cargo.toml" in all_paths
      Source annotation: append "(pyproject.toml)" or "(Dockerfile)" etc. for clarity.

    _build_module_summary(key_file_paths: list[str]) -> list[tuple[str, str]] — for each path, guess purpose from the path string (no file reads, pure heuristic): return list of (path, role_str) pairs. Examples: path ending in "main.py" or "app.py" -> "Application entry point"; path starting with "backend/" -> "Backend service module"; path starting with "frontend/" -> "Frontend source file"; path containing "/tests/" or "/test_" -> "Test module". Catch-all: "Source file".

    _build_markdown(owner: str, repo: str, stack: list[str], tree: str, key_file_contents: dict[str, str], module_summary: list[tuple[str, str]], files_read: int) -> str — assemble the final markdown string with exactly these sections in order:
      1. "# Codebase Snapshot: {owner}/{repo}"
      2. "_Generated {date} by Jarvis. Do not edit manually — overwritten on each scan._"
      3. "## Tech Stack" — bulleted list from stack; "No manifest files detected." if empty
      4. "## Directory Structure" — fenced code block containing tree string
      5. "## Key Files" — for each entry in key_file_contents: "### {filename}\n> {content[:MAX_FILE_CHARS]}"
      6. "## Module Summary" — markdown table with Path and Role columns from module_summary
      7. Footer: "_Files read: {files_read}. Scan completed: {iso_timestamp}_"

    Main coroutine run() — async, uses httpx.AsyncClient with a single shared session for all calls within the scan:

      Step 1: Parse owner/repo from github_repo slug (return early if invalid — no exception raised).

      Step 2: Build GitHub API headers:
        {
          "Authorization": f"Bearer {github_token}",   # T-18-01: never log this header
          "Accept": "application/vnd.github+json",
          "X-GitHub-Api-Version": "2022-11-28",
        }

      Step 3: async with httpx.AsyncClient(headers=headers, timeout=30.0) as client:

      Step 4: GET {GITHUB_API_BASE}/repos/{owner}/{repo} to get default_branch. On non-200: log warning and return.

      Step 5: GET {GITHUB_API_BASE}/repos/{owner}/{repo}/git/trees/HEAD?recursive=1. On non-200: log warning and return. If response JSON has truncated=true: log warning "Trees API truncated for {owner}/{repo} — snapshot may be incomplete" and continue.

      Step 6: Extract blob paths: [item["path"] for item in data["tree"] if item.get("type") == "blob"].

      Step 7: selected = _select_key_files(all_paths).

      Step 8: For each path in selected — GET {GITHUB_API_BASE}/repos/{owner}/{repo}/contents/{path}. On 404/403: log warning and skip (continue). On 200: base64-decode response JSON "content" field; decode bytes to str with errors="replace"; truncate to MAX_FILE_CHARS; store in key_file_contents dict. Collect manifest paths (pyproject.toml, requirements.txt, package.json) into manifest_contents for stack detection.

      Step 9: stack = _detect_tech_stack(all_paths, manifest_contents).

      Step 10: tree = _build_ascii_tree(all_paths).

      Step 11: module_summary = _build_module_summary(list(key_file_contents.keys())).

      Step 12: markdown = _build_markdown(owner, repo, stack, tree, key_file_contents, module_summary, len(key_file_contents)).

      Step 13 — idempotent PUT: GET {GITHUB_API_BASE}/repos/{owner}/{repo}/contents/.hermes/codebase.md to fetch existing sha (status 200 -> sha = resp.json()["sha"]; otherwise sha = None). This is required by GitHub API to update an existing file (T-18-03).

      Step 14 — PUT {GITHUB_API_BASE}/repos/{owner}/{repo}/contents/.hermes/codebase.md with JSON body:
        {
          "message": "chore: update codebase snapshot [jarvis-scan]",
          "content": base64.b64encode(markdown.encode()).decode(),  # T-18-03: base64 markdown only
          "branch": default_branch,
        }
        Add "sha": existing_sha to body if existing_sha is not None.
        On non-2xx response: log warning with status code and raise (so caller can set PipelineState to error).

      Step 15: log info "Codebase snapshot committed to {owner}/{repo}/.hermes/codebase.md (files_read={N})".

    Module docstring must reference T-18-01, T-18-02, T-18-03. Do NOT log the Authorization header value or any portion of github_token at any log level.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| codebase_scan_service → GitHub API | github_token in Authorization header; response paths and content treated as data |
| GitHub API response → markdown output | Tree paths from GitHub treated as data (display only, no eval/exec); content truncated at 2000 chars |
| markdown → GitHub Contents API PUT | Content base64-encoded; no credential values embedded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Information Disclosure | codebase_scan_service.run() — logging | mitigate | github_token never passed to logger.*; Authorization header value never logged at any level; only owner/repo and file count logged |
| T-18-02 | Tampering | _select_key_files, _build_ascii_tree — path handling | mitigate | Tree paths from GitHub API treated as display strings only; no path is eval'd, exec'd, or used in os.path.join for local filesystem access; pathlib.PurePosixPath used for extension/component parsing |
| T-18-03 | Information Disclosure | _build_markdown → PUT body | mitigate | PUT body contains only base64-encoded markdown; no credential values, no decrypted tokens, no env var values embedded in markdown content |
| T-18-04 | Denial of Service | Trees API response — large repos | accept | MAX_FILES=25 cap bounds memory and request count; 2000-char truncation bounds per-file memory; truncated=true logged as warning and scan continues |
| T-18-SC | Tampering | No new package installs | accept | No new pip packages added; httpx already in requirements.txt |
