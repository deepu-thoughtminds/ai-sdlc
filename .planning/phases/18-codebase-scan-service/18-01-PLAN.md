  <action>
Create backend/services/codebase_scanner.py. The module docstring must cite SCAN-02, SCAN-03 and list all threat mitigation tags (T-18-01 through T-18-03). Export one public function: scan_and_commit_codebase(github_repo: str, github_token: str) -> str.

Implementation steps inside scan_and_commit_codebase():

STEP 1 — Clone:
Call clone_repository(github_repo, github_token) from services.repo_clone. This returns a ClonedRepo(workspace_path, owner, repo). Wrap all subsequent steps in try/finally where the finally block always calls shutil.rmtree(workspace_path, ignore_errors=True). (T-18-03: prevents workspace accumulation on failure)

STEP 2 — Directory tree:
Use os.walk(workspace_path) to build a printable tree string. Skip hidden directories starting with "." (e.g. .git) entirely — do not descend into them. For each directory entry, emit "  " * depth + dir_name + "/" and for each file emit "  " * depth + filename. Collect lines in a list; join to a single string. This is always included in the output (SCAN-02: directory tree is always included).

STEP 3 — Identify targeted key files:
Walk the workspace with os.walk but apply these rules to collect a targeted_files list of absolute paths:
  - Root level only: README.md, README.rst, README.txt (case-insensitive match on stem + ext)
  - Root level only: package.json, pyproject.toml, setup.py, setup.cfg, Cargo.toml, go.mod
  - Root level only: requirements.txt and any file matching requirements*.txt
  - Root level only: Dockerfile, docker-compose.yml, docker-compose.yaml
  - Root level only: any file whose name starts with "index." (index.ts, index.js, index.py, etc.)
  - Top-level package directories only (depth=1 relative to workspace root): __init__.py files
  - "Root level" means the file's parent is workspace_path; "depth=1 packages" means parent's parent is workspace_path.
Do NOT read files outside these patterns. This keeps token cost bounded (SCAN-02).

STEP 4 — Read targeted files (cap each at 5000 chars):
For each path in targeted_files (already absolute), open with encoding="utf-8", errors="replace". Read up to 5001 chars; truncate at 5000 and append "\n... [truncated]" if longer. Store in a dict: relative_path -> content.

STEP 5 — Detect tech stack:
Derive a list of (language_label, reason) tuples from the presence of files in targeted_files_dict keys (relative paths):
  - "pyproject.toml" or "requirements.txt" present → ("Python", "pyproject.toml / requirements.txt found")
  - "package.json" present → ("Node.js / JavaScript", "package.json found")
  - "Cargo.toml" present → ("Rust", "Cargo.toml found")
  - "go.mod" present → ("Go", "go.mod found")
  - "setup.py" or "setup.cfg" present → ("Python", "setup.py/setup.cfg found") — deduplicate with pyproject
If no indicator found: [("Unknown", "no standard config files detected")]
Deduplicate by label.

STEP 6 — Build .hermes/codebase.md content:
Generate a markdown string with the following structure (no LLM involved):

```
# Codebase Snapshot

_Generated: {datetime.utcnow().strftime("%Y-%m-%dT%H:%M:%SZ")} by Jarvis — do not edit manually_

## Directory Tree
```
{tree_string}
```

## Detected Tech Stack
{for each (label, reason): "- **{label}**: {reason}"}

## Key Files Read
| File | Size (chars) |
|------|-------------|
{for each relative_path: "| {relative_path} | {len(content)} |"}

## Module Summary
{for each top-level directory found during os.walk depth=1: read first 3 non-empty lines of its __init__.py if present, else emit "- **{dirname}/**: (no __init__.py)"}
```

The closing ``` for Directory Tree must be a real fenced code block in the markdown — use triple backticks as literal characters in the string.

STEP 7 — Write the file to workspace:
Build the output path: pathlib.Path(workspace_path) / ".hermes" / "codebase.md". Create the .hermes directory: output_path.parent.mkdir(parents=True, exist_ok=True). Write the content string to output_path with encoding="utf-8".

STEP 8 — Git configure, add, commit, push to main:
Use _run_git() imported from services.pr_creator. Pass github_token to every _run_git() call so errors are token-scrubbed (T-18-01: token never logged).

  _run_git(["config", "user.email", "jarvis@ai-sdlc.bot"], cwd=workspace_path, github_token=github_token)
  _run_git(["config", "user.name", "Jarvis Bot"], cwd=workspace_path, github_token=github_token)
  _run_git(["add", ".hermes/codebase.md"], cwd=workspace_path, github_token=github_token)
  _run_git(["commit", "-m", "chore: update codebase snapshot [jarvis-scan]"], cwd=workspace_path, github_token=github_token)

For push, build the authenticated URL using oauth2: scheme (not x-access-token: — see repo_clone.py comment about PAT write-access check):
  github_host = os.environ.get("GITHUB_HOST", "github.com")
  push_url = f"https://oauth2:{github_token}@{github_host}/{owner}/{repo}.git"
  _run_git(["push", push_url, "HEAD:main"], cwd=workspace_path, github_token=github_token)

T-18-02: subprocess.run uses list args (enforced by _run_git — never shell=True).

STEP 9 — Extract and return commit SHA:
After the commit step succeeds, run:
  result = _run_git(["rev-parse", "HEAD"], cwd=workspace_path, github_token=github_token)
  commit_sha = result.stdout.strip()
Return commit_sha. Log at INFO: "Committed codebase snapshot for {owner}/{repo} sha={commit_sha}" — no token in this log line (T-18-01).

Module-level logger: logger = logging.getLogger(__name__)
Imports needed: logging, os, pathlib, shutil, datetime from datetime, subprocess (via _run_git), ClonedRepo from services.repo_clone (clone_repository), _run_git from services.pr_creator.

Do NOT import or call any LLM client. Do NOT use shell=True anywhere.
