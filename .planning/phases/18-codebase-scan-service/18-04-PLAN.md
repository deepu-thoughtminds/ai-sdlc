<objective>
Fix the silent-failure path in codebase_scan_service.run() so that GitHub API failures cause
PipelineState to transition to 'error' rather than remaining stuck at 'running'.

Purpose: UAT test 5 confirmed that a 401 from GitHub leaves PipelineState.status='running'
indefinitely. The root cause is that run() returns None on non-200 repo metadata and trees API
responses, so the except Exception block in _run_scan_background() never fires.

Output:
- codebase_scan_service.py: two `return` statements on GitHub API failure paths replaced with
  `raise RuntimeError(...)`.
- test_codebase_scan_service.py: two new regression tests (repo metadata 401 raises, trees API
  403 raises). The existing test_run_invalid_repo_returns_early (invalid slug → return) is
  unaffected because that path precedes any API call.
</objective>
