<objective>
Create a standalone, reusable codebase-snapshot reader (`get_codebase_snapshot()`) that fetches `.hermes/codebase.md` via the GitHub Contents API and degrades gracefully — returning `None` on any failure (404, network error, bad response) rather than raising or surfacing an empty error to the user (SNAPSHOT-02).

Purpose: Phase 18 built the writer (`codebase_scan_service.run()`); no reader exists anywhere in the codebase yet. This reader has no consumer in this phase — Phase 20 (`describe_pipeline.py`) and Phase 21 (`architecture_pipeline.py`) will be its first callers. Build it now as a well-tested, importable function so those phases can wire it in directly without redesigning the read path.
Output: `backend/services/codebase_snapshot_reader.py` with `get_codebase_snapshot(github_repo: str, github_token: str) -> str | None`, plus full respx-mocked test coverage in `backend/tests/test_codebase_snapshot_reader.py`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| codebase_snapshot_reader.py -> GitHub Contents API | Outbound authenticated HTTP call; token placed only in Authorization header |
| codebase_snapshot_reader.py -> future callers (Phase 20/21) | Return value (str | None) is the only contract; no exception type is ever part of the public contract |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-19-04 | Information Disclosure | codebase_snapshot_reader.py get_codebase_snapshot() | mitigate | github_token placed only in the `Authorization` header dict; all `logger.warning`/`logger.info` calls interpolate only `owner`, `repo`, and the caught exception object `exc` (never the token); enforced by test_get_codebase_snapshot_token_never_in_logs_or_exceptions |
| T-19-05 | Denial of Service | codebase_snapshot_reader.py httpx call | mitigate | `httpx.AsyncClient(..., timeout=15.0)` bounds the request; any timeout/connection error is caught by the blanket `except Exception` and converted to `None`, never propagated to the caller |
| T-19-06 | Repudiation / silent failure surfaced as error | codebase_snapshot_reader.py 404 handling | mitigate | 404 is treated as a normal, expected, non-error outcome (file simply doesn't exist yet) — logged at INFO not WARNING, and returns the same `None` sentinel as other failure modes so callers have one uniform fallback path |
</threat_model>
