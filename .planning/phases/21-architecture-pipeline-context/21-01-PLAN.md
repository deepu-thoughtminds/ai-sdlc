<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| architecture_pipeline.run() → GitHub API | github_token is decrypted credential; must not appear in logs or prompts |
| GitHub API → codebase_snapshot_reader | Response content is untrusted user-controlled repo content; truncated before inclusion in LLM prompt |
| LLM prompt | Contains issue summary/description from Jira and codebase snapshot — no credential values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | Information Disclosure | architecture_pipeline.run() logging | mitigate | decrypt github_token and github_repo using try/except; log only issue_key in all logger calls; decrypted values never passed to any log statement |
| T-21-02 | Tampering / DoS | LLM prompt construction | mitigate | snapshot[:8000] truncation prevents token overrun; None snapshot handled with fallback text "(no codebase context available)" |
| T-21-03 | Spoofing | codebase snapshot content | accept | Snapshot sourced from project's own GitHub repo via authenticated token; low risk of injection beyond prompt length |
| T-21-SC | Tampering | npm/pip/cargo installs | accept | No new package installations in this plan — uses existing codebase_snapshot_reader import only |
</threat_model>
