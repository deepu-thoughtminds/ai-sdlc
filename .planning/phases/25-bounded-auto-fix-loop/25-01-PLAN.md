    Main function run_auto_fix_loop(unit_test_results, workspace_path, issue_key, github_repo, github_token, state_row, db) -> tuple[list[TestResult], str | None]:

      1. Filter failing: [r for r in unit_test_results if r.returncode != 0 and not r.timed_out]
         If empty, return (unit_test_results, None) immediately.

      2. workspace_root = pathlib.Path(workspace_path).resolve()
         accumulated_changes: list[FileChange] = []
         pr_url: str | None = None
         prev_fingerprints: frozenset[str] | None = None
         current_results = list(unit_test_results)

      3. for attempt in range(1, MAX_ATTEMPTS + 1):
           a. Update state_row.qa_attempt = attempt; db.commit()  — AUTOFIX-04 tracking

           b. failing = [r for r in current_results if r.returncode != 0 and not r.timed_out]
              If not failing: break (all tests now pass)

           c. Non-progress check (AUTOFIX-02):
              current_fp = _fingerprint(failing)
              if prev_fingerprints is not None and current_fp == prev_fingerprints:
                  logger.info("Non-progress detected at attempt %d — terminating early", attempt)
                  break
              prev_fingerprints = current_fp

           d. Build fix prompt per failing result (AUTOFIX-01):
              For each r in failing:
                stdout_snip = (r.stdout or "")[:2000]   # T-25-03
                stderr_snip = (r.stderr or "")[:2000]   # T-25-03
                prompt section: "Tool: {r.tool}\nSTDOUT (truncated):\n{stdout_snip}\nSTDERR (truncated):\n{stderr_snip}\n"
              Combine all failing sections into one prompt.
              Include instruction: "Fix the source files (NOT test files) to make the above tests pass.
              For each source file you modify, use the ### FILE: path/to/source.py format.
              IMPORTANT: Do NOT modify any files under tests/ — only fix application source code."

           e. response = route_request("autofix", prompt)
              new_changes = _parse_file_changes(response.content)

           f. Apply path guards (T-25-01, T-25-02):
              valid_changes: list[FileChange] = []
              for change in new_changes:
                  try:
                      resolved = (workspace_root / change.path).resolve()
                  except Exception:
                      continue
                  # T-25-01: path traversal guard
                  if not str(resolved).startswith(str(workspace_root) + "/"):
                      logger.warning("Auto-fix: rejected path-traversal attempt: %s", change.path)
                      continue
                  # T-25-02: reject test file modification
                  rel = str(pathlib.Path(change.path))
                  if rel.startswith("tests/") or rel.startswith("tests\\"):
                      logger.warning("Auto-fix: rejected test file modification: %s", change.path)
                      continue
                  valid_changes.append(change)

           g. If valid_changes:
                # Write changes to disk (for re-run)
                for change in valid_changes:
                    resolved_path = (workspace_root / change.path).resolve()
                    resolved_path.parent.mkdir(parents=True, exist_ok=True)
                    resolved_path.write_text(change.content, encoding="utf-8")
                accumulated_changes.extend(valid_changes)

           h. Re-run only the failing tests (AUTOFIX-01):
              failing_tool_names = [r.tool for r in failing]
              current_results = _rerun_failing_tests(failing_tool_names, workspace_path)

      4. After loop — open PR if any fix was applied (AUTOFIX-03):
         If accumulated_changes:
             branch_name = f"jarvis/qa-fix-{issue_key}"
             pr_title = f"fix(qa): auto-fix {issue_key} — attempt {attempt}/{MAX_ATTEMPTS}"
             pr_body = (
                 f"Automated QA fix generated by Jarvis for {issue_key}.\n\n"
                 f"Applied {len(accumulated_changes)} source file change(s) across "
                 f"{min(attempt, MAX_ATTEMPTS)} attempt(s).\n\n"
                 "This PR was created by the Jarvis auto-fix loop — review before merging."
             )
             try:
                 pr = apply_commit_push_and_open_pr(
                     workspace_path=workspace_path,
                     github_repo=github_repo,
                     github_token=github_token,
                     issue_key=issue_key,
                     file_changes=accumulated_changes,
                     branch_name=branch_name,
                     pr_title=pr_title,
                     pr_body=pr_body,
                 )
                 pr_url = pr.html_url
             except Exception as exc:
                 logger.warning("Auto-fix: PR creation failed for %s: %s", issue_key, exc)

      5. return (current_results, pr_url)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| freellmapi → auto_fix_loop | LLM-generated text crosses into filesystem write path via _parse_file_changes |
| workspace_root → host filesystem | FileChange.path must not escape the cloned repo directory |
| tests/ prefix → application source | LLM must not be allowed to modify test files to make tests pass trivially |
| auto_fix_loop → GitHub remote | All commits must go to jarvis/qa-fix-{issue_key} branch, never main |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Tampering | auto_fix_loop._apply_path_guards | mitigate | resolve (workspace_root / change.path).resolve(); reject if not under workspace_root + "/"; log + skip per file; mirrors T-24-01 guard in qa_pipeline.py |
| T-25-02 | Tampering | auto_fix_loop._apply_path_guards | mitigate | reject FileChange where rel path starts with "tests/" or "tests\"; check BEFORE writing to disk; log + skip; LLM is instructed in prompt to fix only source (defence in depth) |
| T-25-03 | Information Disclosure | auto_fix_loop.run_auto_fix_loop prompt builder | mitigate | truncate stdout and stderr to 2000 chars each before interpolating into fix prompt; prevents internal path / secret leakage via oversized LLM context |
| T-25-04 | Elevation of Privilege | auto_fix_loop → pr_creator | mitigate | AUTOFIX-03 enforced: branch_name always "jarvis/qa-fix-{issue_key}"; apply_commit_push_and_open_pr targets base_branch="main" via PR only — no direct push to main occurs |
| T-25-05 | Tampering | auto_fix_loop._fingerprint (non-progress detection) | mitigate | fingerprints computed from raw (untruncated) stderr+stdout before truncation; compared as frozenset for order-independence; if prev_fingerprints == current_fp → loop terminates (AUTOFIX-02) |
| T-25-SC | Tampering | no new package installs in this phase | accept | auto_fix_loop.py reuses only already-installed imports (hashlib stdlib, pathlib stdlib, services.*); no new pip/npm installs required |
</threat_model>
