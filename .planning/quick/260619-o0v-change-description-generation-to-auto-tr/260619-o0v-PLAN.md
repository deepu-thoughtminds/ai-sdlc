<objective>
Change description generation from @jarvis describe (comment-triggered) to auto-trigger on Jira
Story creation. After generation, post the draft as a comment asking for @jarvis approve story
description. On that mention, update the Jira ticket description field. Also rename the architecture
approval trigger to @jarvis approve architecture, making all approvals mention-based.

Purpose: Remove manual trigger friction — teams should not need to @jarvis describe after creating
a story; it happens automatically. Approvals become explicit and intent-clear via distinct mention
phrases rather than generic LGTM keywords.

- New /webhook/jira-issue endpoint handling issue_created events
- Updated mention_parser: describe removed, approve added with sub-command capture
- Updated approval_detector: detects @jarvis approve story description and @jarvis approve architecture
- Updated webhook.py: routes approve mentions to approval_detector
- Updated tests: 3 test files reflecting the new behavior
</objective>
