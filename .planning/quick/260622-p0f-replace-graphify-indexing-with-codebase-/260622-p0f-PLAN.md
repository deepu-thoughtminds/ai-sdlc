# Quick Task 260622-p0f: Replace graphify with codebase-memory-mcp

## Context

The dev pipeline (`backend/services/dev_pipeline.py`) runs `claude_code_executor.py` when `CLAUDE_API_KEY` is set. That executor currently instructs claude CLI to:
1. Run `/graphify update .` — LLM-based codebase indexing (slow, expensive)
2. Run `/gsd-graphify plan` — plan code changes using graphify index
3. Run `/gsd-quick` — implement changes

The goal is to replace steps 1–2 with `codebase-memory-mcp`, a static binary MCP server that indexes codebases without LLM. The binary is installed as npm package `codebase-memory-mcp@0.8.1`. It runs as a stdio MCP server that the claude CLI picks up via `~/.claude/.mcp.json`.

## Task 1: Update backend/Dockerfile to install codebase-memory-mcp

**Files:** `backend/Dockerfile`

**Action:**
- Add Node.js installation (nodejs + npm via apt)
- Install `codebase-memory-mcp` via `npm install -g codebase-memory-mcp`
- Write `~/.claude/.mcp.json` so the claude CLI (when run by claude_code_executor.py) uses codebase-memory-mcp as a stdio MCP server

**Verify:** Dockerfile has Node.js install, npm install of codebase-memory-mcp, and .mcp.json creation step
**Done:** `backend/Dockerfile` builds without error and has all three additions

## Task 2: Update claude_code_executor.py prompt

**Files:** `backend/services/claude_code_executor.py`

**Action:**
- Remove step 1 (`/graphify update .`) from the prompt
- Remove step 2 (`/gsd-graphify plan`) from the prompt
- Replace with: index the codebase using codebase-memory-mcp's `index_repository` tool, then use `search_graph`/`get_code_snippet` tools to understand relevant code, then use `/gsd-quick` to plan and implement the story
- Update the module docstring to reflect the new approach

**Verify:** Prompt no longer contains graphify/gsd-graphify; contains codebase-memory-mcp indexing + gsd-quick instructions
**Done:** `claude_code_executor.py` docstring and prompt updated; no graphify references remain
